## 3. 已冻结的协作与技术要求
- 代码示例统一使用**更详细的中文注释**
  - 不只说明“做什么”
  - 还要说明“为什么这样写”
  - 以及“后续应该改哪里”
- 开发指导模式保持：
  - **一次只推进一步**
  - 不要一次性展开全部步骤
  - 每完成一步，再进入下一步
- 输入方案固定为：
  - **Input Manager (Old)**
  - 当前版本不切换到 New Input System
- 敌人追踪目标：
  - 不在 `Update` 中频繁使用 `Find / FindWithTag`
  - 当前优先用**手动拖引用**或**启动时一次性缓存**
  - 后续刷怪系统阶段再统一改为全局缓存方案
- 当前近战怪不继续使用：
  - **身体碰撞 = 伤害判定**
- 当前建议方案：
  - **身体碰撞盒 + 独立 AttackRange Trigger + 低摩擦材质**
- 项目范围保持作品集垂直切片目标：
  - 不扩张为开放大地图
  - 不扩张为经营系统
  - 不扩张为多角色
  - 不扩张为大型剧情
